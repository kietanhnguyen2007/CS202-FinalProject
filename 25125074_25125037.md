# II. EXECUTIVE SUMMARY AND PROJECT OBJECTIVES

## Project Overview

This project proposes the development of a multiplayer action-platformer inspired by Apple Knight using C++ and Raylib.

The game supports both single-player and multiplayer gameplay through Local Area Network (LAN) and Radmin VPN.

Players cooperate to explore levels, defeat enemies, solve puzzles, collect treasures, and challenge powerful bosses.

The project emphasizes:

* Object-Oriented Programming principles
* Multiplayer networking
* Modular software architecture
* Performance optimization
* Real-time synchronization
* Scalable game systems

---

## Primary Objectives

### Technical Objectives

* Build a reusable game engine architecture.
* Implement multiplayer networking using ENet.
* Support LAN and Radmin VPN connectivity.
* Maintain stable 60 FPS performance.
* Apply advanced design patterns.
* Minimize runtime memory allocation.

### Gameplay Objectives

* Create engaging platforming mechanics.
* Implement co-op multiplayer gameplay.
* Design boss battles and enemy AI.
* Develop inventory and loot systems.
* Introduce cooperative puzzle mechanics.

## Object-Oriented Entity Hierarchy

| Class      | Responsibility             |
| ---------- | -------------------------- |
| Entity     | Base game object           |
| Character  | Movement and combat        |
| Player     | Human-controlled character |
| Enemy      | AI-controlled character    |
| Boss       | Advanced enemy             |
| Item       | Collectible objects        |
| Projectile | Combat projectiles         |

### Entity

```cpp
virtual void Update(float dt) = 0;
virtual void Render() = 0;
virtual Rectangle GetBounds() const;
```

### Character

```cpp
virtual void Move(Vector2 direction);
virtual void Jump();
virtual void Attack();
virtual void TakeDamage(int amount);
```

### Player

```cpp
void Update(float dt) override;
void Render() override;
void UseSkill();
```

# VII. DATA PROCESSING CHALLENGES AND SOLUTIONS

## 1. Memory Fragmentation

### Problem

Frequent creation of:

* Projectiles
* Effects
* Particles

### Solution

Object Pooling

```cpp
template <typename T, size_t PoolSize>
class ObjectPool
{
};
```

Benefits:

* Predictable memory usage
* Faster allocation
* Better performance

---

## 2. Collision Detection Complexity

### Problem

Large numbers of entities increase collision checks.

### Solution

Quadtree Spatial Partitioning

Benefits:

* Faster collision detection
* Improved scalability
* Reduced CPU usage

---

## 3. Network Bandwidth Constraints

### Solution A

Dead Reckoning

Benefits:

* Smoother movement
* Lower network traffic

### Solution B

Interest Management

Benefits:

* Reduced unnecessary updates
* Better scalability
